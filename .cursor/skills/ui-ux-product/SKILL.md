# UI UX Product

## Scope

Apply for any UI structure, interaction, or copy change.

## Do

- Follow `docs/design/DESIGN.md` for palette, spacing, and hierarchy.
- Prioritize mobile-first ergonomics and readable density.
- Keep CTAs explicit and state feedback immediate.
- Preserve consistency of component behaviors across coach/client surfaces.

## Dont

- Do not overuse accent color or visual noise.
- Do not introduce inaccessible contrast or ambiguous disabled states.
- Do not mix unrelated interaction models in neighboring screens.

## Checklist

- [ ] Visual hierarchy and spacing match LoBoost conventions.
- [ ] Empty/loading/error states are intentional.
- [ ] Interaction labels are explicit and in-product tone.
- [ ] UX flow has no dead-end step.

## Examples

- Primary action uses accent style and clear outcome label.
- Status blocks (warning/error/completed) map to documented semantic colors.
